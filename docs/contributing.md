# Development Rules

The project is developed incrementally.

Architecture is designed before implementation.

Every subsystem must have a single responsibility.

Avoid temporary fixes.

Avoid duplicated systems.

Delete obsolete code instead of preserving compatibility.

Maintain a single source of truth for every piece of data.

Build frequently.

Keep modules independent.

Folders are created only when their first implementation begins.

Every new feature should preserve existing architecture rather than redesign it.

When in doubt, prefer simplicity over cleverness.
